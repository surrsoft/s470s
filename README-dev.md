# s470s — Developer Guide

## счётчик F_F

- 56

## понятия

- `!!s470s-select-mode!!`: режим выбора
- `!!s470s-search-mode!!`: режим поиска — активируется когда поле F8F не пустое; в этом режиме список F17F показывает результаты поиска по всем заметкам (не только текущего уровня иерархии); поиск выполняется по мере ввода с учётом настроек фильтрации (F9F–F12F) и сортировки (F14F); при очистке поля F8F список возвращается в обычный режим
- `F44F(n,t)`: означает что заметка `n` помечена тегом `t` (если `t` это тег) либо группой тегов `t` (если `t` это группа тегов)

## Установка для разработки

1. Склонируйте или скачайте этот репозиторий
2. Откройте Chrome/Edge и перейдите на страницу расширений:
   - Chrome: `chrome://extensions/`
   - Edge: `edge://extensions/`
3. Включите "Режим разработчика" (Developer mode)
4. Нажмите "Загрузить распакованное расширение" (Load unpacked)
5. Выберите папку с расширением

## Технические детали

- **Manifest Version**: 3
- **Permissions**:
  - `storage` — для сохранения заметок
  - `clipboardWrite` — для копирования в буфер обмена
  - `sidePanel` — для открытия боковой панели в Chrome
- **Host Permissions**: `https://*.supabase.co/*` — для API-запросов к Supabase; `https://geocoding-api.open-meteo.com/*`, `https://api.open-meteo.com/*` — для погоды
- **Хранилище**: Chrome Storage API (локальное)
- **Синхронизация**: Supabase (PostgreSQL + Realtime WebSocket), опционально
- **Погода (F28F)**: [Open-Meteo](https://open-meteo.com/) — бесплатный open-source API, без ключа; геокодинг → `geocoding-api.open-meteo.com`, прогноз → `api.open-meteo.com`; кэш 30 минут
- **Side Panel**: в Chrome 114+ расширение открывается как боковая панель; в других браузерах — как popup

## Настройка Supabase (для разработки)

1. Создайте проект на [supabase.com](https://supabase.com)
2. Перейдите в **SQL Editor** и выполните:

```sql
create table notes (
  id           uuid primary key default gen_random_uuid(),
  user_id      uuid references auth.users not null,
  local_id     text not null,
  copy_text    text not null,
  description  text,
  url          text,
  img          text, 
  parent_id        text,             -- ссылка на local_id родительской заметки
  parent_ids_other text[],           -- массив local_id дополнительных родителей (симлинки)
  is_fast_copy boolean not null default false,
  show_time    boolean not null default false, -- F27F: отображать текущее время
  timezone     text,                           -- F27F: IANA часовой пояс (e.g. Europe/Moscow)
  show_weather boolean not null default false, -- F31F
  weather_city text,                           -- F29F
  weather_type text not null default 'min',    -- F28F, F30F, и т.д.
  tags         jsonb,                          -- F45F: массив групп тегов [['tag1'], ['tag2', 'tag3']]
  is_tag       boolean not null default false, -- F43F: является ли заметка тегом
  tag_color    text,                           -- F48F: цвет тега в HEX
  ls_path      text,                           -- F50F: путь к значению в localStorage (формат key.path)
  "order"      integer not null default 0,
  created_at   bigint not null,
  updated_at   bigint not null,
  date_actual  timestamp with time zone,       -- дата актуальности
  deleted_at   timestamp with time zone,       -- soft-delete: when moved to trash (null = active)
  unique(user_id, local_id)
);

alter table notes enable row level security;

create policy "Users manage own notes" on notes
  for all using (auth.uid() = user_id);
```

3. В разделе **Authentication → Email** включите "Enable Email OTP" и отключите "Confirm email" (если нужен вход без подтверждения)
4. Скопируйте **Project URL** и **Anon Key** из **Settings → API**
5. Вставьте их в настройках расширения (кнопка ⚙)

## Основные функции

### popup.js
- `loadNotes()` — загрузка заметок из хранилища
- `saveNotes()` — сохранение заметок в хранилище
- `render()` — отрисовка списка заметок
- `addNote()` — создание новой заметки
- `updateNote()` — обновление существующей заметки
- `deleteNote()` — удаление заметки
- `copyToClipboard()` — копирование текста в буфер обмена
- `loadFontSize()` — загрузка и применение сохранённого размера шрифта
- `applyFontSize()` — применение текущего размера к списку заметок
- `loadTheme()` — загрузка и применение сохранённой темы
- `applyTheme()` — применение текущей темы (light/dark) к body
- `serverRowToNote(row)` — конвертация строки Supabase в формат заметки
- `initSync()` — инициализация синхронизации при загрузке
- `runFullSync()` — полная двусторонняя синхронизация с Supabase
- `scheduleSync()` — отложенная отправка изменений (debounce 1.5с)
- Функции drag & drop для изменения порядка заметок

### sync/supabase.js
- `setConfig(config)` — установка конфигурации клиента
- `setSession(session)` — установка сессии аутентификации
- `signIn(email)` — отправка OTP-кода на email
- `verifyOtp(email, token)` — верификация кода, получение сессии
- `signOut()` — выход из аккаунта
- `fetchNotes()` — получение заметок из Supabase
- `upsertNote(note)` / `upsertNotesBatch(notes)` — сохранение заметок
- `deleteNote(localId)` — удаление заметки
- `subscribeRealtime(handlers)` — подписка на изменения в реальном времени
- `deleteNoteRemote(localId)` — удаление заметки из Supabase (переименовано из `deleteNote` во избежание конфликта с UI-функцией)

## функционалы

### F42F, теги, tags

- заметка может быть тегом; у такой заметки должен быть включён чекбокс F43F
- к любой заметке можно добавить тег или группу тегов
- если к заметке привязан тег или группа тегов, то на заметке отображается название тега или группы тегов
- если пользователь удаляет заметку-тег, либо отключает для неё свойство тега, то для всех заметок к которым этот тег привязан, выполняются соответствующие операции технической отвязки тега
